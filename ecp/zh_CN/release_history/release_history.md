# 发版历史

## v2.7.0
发布日期: 2025-11-07

### 增强

ECP v2.7.0 引入强大的数据集成模块。该模块基于统一命名空间（UNS）和SparkplugB标准，旨在打通从边缘到云端的数据链路，实现工业设备、应用与AI的无缝融合，为智能制造提供坚实的数据基础。

1、推出全新的数据集成功能，构建基于统一命名空间（UNS）的工业AI数据中枢

- 数据集成->总览页面: 树状层级结构: 新增“总览”页面，通过UNS+SparkplugB数据建模，以层级分明的树状结构展示NeuronEX的边缘节点、设备及数据点位，完整还原工业现场的物理层级与业务逻辑。
	
	- 树状层级结构: 新增“总览”页面，通过UNS+SparkplugB数据建模，以层级分明的树状结构展示NeuronEX的边缘节点、设备及数据点位，完整还原工业现场的物理层级与业务逻辑。

    - 实时状态监控: 在UNS树中，用户可直观查看各边缘节点和设备的实时在线/离线状态，并获取节点、设备和点位的数量概览。

    - 别名与层级管理: 支持为各层级节点、设备自定义别名，方便用户以符合业务习惯的方式识别和管理。同时，支持删除离线节点，保持数据拓扑整洁。

- 集成时序数据存储与分析

	- 内置Datalayers数据库: ECP 默认集成 Datalayers 时序数据库，支持10万点位数据的秒级存储。同时支持连接外部Datalayers集群，满足百万级点位、长周期存储的高需求。

	- SQL数据分析: 提供全新的“数据分析”页面，用户可对存储于 Datalayers 的历史数据进行SQL查询与分析。该功能与 NeuronEX 3.6.0 的数据分析模块体验一致，支持智能 SQL 输入、结果图表化展示。

	- AI辅助分析: 集成AI能力，可通过自然语言生成复杂的SQL查询，帮助用户快速洞察数据价值。

- AI驱动的智能运维

    - AI运维助手: 新增“AI问答”功能，用户可通过自然语言，结合点位ID或别名，对设备状态和异常提问。

    - 融合知识库: 内置 NeuronEX 运维知识库，结合LLM大语言模型，帮助用户快速解答关于 NeuronEX 配置、使用和故障排查的相关问题。

- 事件查询与追溯

    - 提供“事件查询”功能，支持根据边缘节点或设备UID、时间范围等条件，查询其上下线的历史事件记录，便于问题追溯与审计。

- 点位反控

  	- 针对通过SparkplugB接入的点位，提供标准化的反控机制，确保数据闭环，满足远程控制与调度需求。

  	- 支持通过 ECP UI进行单个数据点位的写入控制。

  	- 支持通过 API / MQTT 进行单个或多个点位的写入控制。

2、边缘服务列表页面，支持检索所有 NeuronEX 中的南北向驱动、规则等元数据。

3、ECP 链路追踪功能支持 License 控制追踪 TPS 功能。

4、ECP 默认不启动 telegraf 服务，根据 emqx_ecp.conf 中 `TELEGRAF_ENABLE` 来决定是否启动，默认为 `false` 。
